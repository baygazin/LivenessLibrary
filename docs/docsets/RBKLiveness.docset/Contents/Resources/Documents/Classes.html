  • RBKCameraController

    RBKCameraController: A UIViewController subclass specifically designed to handle the complexities of managing camera functionalities within an iOS application. This class integrates directly with the AVFoundation framework to manage an AVCaptureSession for capturing video and photo content. It provides a robust infrastructure for configuring, displaying, and controlling the camera capture session.

    Key Functionalities:

    • Manages an AVCaptureSession to coordinate the data flow from the camera hardware to the application.
    • Utilizes an AVCaptureVideoPreviewLayer to display a live camera feed within the app’s UI, offering users a real-time view of what the camera sees.
    • Supports custom configuration of camera settings and outputs, making it adaptable for various capture needs like high-resolution photos or continuous video recording.

    Example Usage:

    • In a video conferencing app, RBKCameraController can be used to manage the video feed from the user’s camera, ensuring high-quality video capture and display.
    • For applications that require document scanning, this controller can manage the camera to capture high-resolution images of documents, adjusting settings for optimal clarity and detail.

    Customization Points:

    • Subclasses can override the setOutputs() method to specify different types of data outputs from the capture session, such as setting up different file formats or resolutions for captured video or photos.
    • The closure didFinishRecording can be customized to handle post-recording tasks, such as processing the video file or updating the UI to reflect that recording has completed.

    This class is essential for any iOS application that requires direct control over camera hardware, providing a flexible and powerful tool for capturing and displaying live video and photos.

  • RBKLivenessCameraController

    RBKLivenessCameraController A specialized controller that extends RBKCameraController, designed to manage and process video and facial detection data for liveness verification within iOS applications. This class leverages AVFoundation for camera management and MLKit for real-time face detection to support a robust and responsive liveness detection system.

    Responsibilities:

    • Manages an AVCaptureSession for capturing live video data.
    • Processes live camera feeds to detect liveness actions such as blinking, smiling, or head movements.
    • Utilizes timers to handle the synchronization of liveness detection events and actions.
    • Communicates with a delegate to provide real-time feedback on the liveness detection status and results.

    Example Usage:

    • Security applications where user verification is required, using live facial features to ensure that the user is physically present.
    • Real-time access control systems where individuals are verified through dynamic liveness checks before granting access.

    Implementation:

    • Initializes and configures the camera capture settings and output processing.
    • Handles asynchronous detection results to optimize the accuracy and efficiency of the system.
    • Maintains state management through properties that track recording status, action sequences, and detection outcomes.

    This class is integral to applications requiring high assurance of user presence and authenticity, making it suitable for sectors like banking, secure access, and remote proctoring.

  • RBKLivenessViewController

    RBKLivenessViewController: A UIViewController subclass that orchestrates the user interface and interaction flows for liveness detection in an iOS application. This class integrates various components including a camera controller, UI elements like labels and activity indicators, and business logic through an associated interactor. It manages the presentation of camera feeds, user prompts (e.g., “Turn your head to the left”), and feedback based on liveness detection results.

    Example Usage:

    • In a banking app, RBKLivenessViewController could be used during a security check to ensure that the account holder is a real person and is present during the transaction.
    • For access control in secure facilities, this controller can manage the process of capturing live facial features to compare with stored biometric data, ensuring that only authorized individuals gain entry.

    Key Components:

    • cameraController: Manages camera setup, video capture, and real-time face detection.
    • titleLabel, descriptionLabel: Display instructions and feedback to the user, guiding them through necessary actions or providing status updates.
    • focusView, arrowsView: Visual cues to direct the user’s gaze or head movement, enhancing the accuracy of liveness detection.
    • activityIndicatorView: Indicates processing or loading states, improving the user interface responsiveness.

    This controller is designed to be embedded within a larger authentication or security workflow, coordinating with other components of the application to provide a seamless and secure user experience.
